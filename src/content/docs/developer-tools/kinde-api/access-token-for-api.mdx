To securely connect to Kinde’s API, you need to obtain an access token. Once you have completed the [connection steps](/developer-tools/kinde-api/connect-to-kinde-api/), you're ready to make an access token request.

## Get the access token (Postman)

We recommend you do this in a non-production environment first.

If you decide to use Postman, we recommended that you set up a Postman environment. Here's some [troubleshooting solutions](/developer-tools/kinde-api/troubleshoot-kinde-api/) in case you need them.

1. Add your Kinde machine to machine [application keys] as environment variables. You might have these copied from when you created the application.

4. Go to the **Authorization** section and set the **Type** to **OAuth 2.0** and set the **Header Prefix** to **Bearer**.
5. In the **Configure New Token > Configuration options** section, set the **Grant Type** to **Client Credentials**.
6. Enter the **Access Token URL** using the domain variable you created above. For example, `{{business_domain}}/oauth2/token`. Note that even if you use a custom domain, the access token URL should still use your `https://<your_subdomain>.kinde.com` domain.

7. Enter the **Client ID** and **Client Secret** using the environment variables you created earlier or by copying them from the Kinde application.
8. Set the **audience** to `{{business_domain}}/api`. To do this:
   - Scroll down click **Advanced**. In the **Token request** section, select the `audience` key and enter the above URL in the **Value** field. Ensure it is being sent in the `body` of the request

12. Select **Proceed**.

14. Next (optional): [Test the connection to Kinde’s API](/developer-tools/kinde-api/troubleshoot-kinde-api/).

## Get the access token (Node.js fetch)

Use the following, but replace `<your_domain>`, `<client_id>` and `<client_secret>` with your own details.

```jsx
	fetch(`${<your_domain>}.kinde.com/oauth2/token`, {
			method: "POST",
			headers: {
				"content-type": "application/x-www-form-urlencoded",
			},
			body: new URLSearchParams({
				audience: "<auth_domain>/api",
				grant_type: "client_credentials",
				client_id: <client_id>,
				client_secret: <client_secret>,
			}),
	})
```

See our [other SDK frameworks](/developer-tools/about/our-sdks/) for additional code snippets.

## Test the various endpoints

You can test the API connection by sending a `GET` request to any Kinde API endpoint. See the [Kinde Management API library](https://kinde.com/api/docs/#kinde-management-api) for options.

Here’s an example in Postman, using the `users listing` endpoint.

1. Create a new GET request.
2. Enter a URL that contains the `/users` endpoint, e.g. `https://<domain>.kinde.com/api/v1/users` .

  src="https://imagedelivery.net/skPPZTHzSlcslvHjesZQcQ/44dd0ac2-b96e-4a91-79dd-4a9f9dda2000/public"

3. Send the request. It should return your users in the body section. If it does, the connection is successful.

<img

4. Repeat from step 2 for any other [Kinde API endpoints](https://kinde.com/api/docs/#kinde-management-api) you want to test.
