Sometimes you'll get an error when setting up your authentication flows. 

Here are some common error codes and troubleshooting steps.

## 578

Handle SAML callback - error validating SAML response

## 780

OAuth 2.0 response failed - redirect ID is null

## 928

Handle Connected App callback - Error getting connected app configuration

## 1004

Initialize SAML redirect - error getting custom SAML config

## 1029

Create org pipeline - assign organization - attempt to assign a new user to an existing Kinde org

## 1061

Handle Kinde to Kinde callback - error with exchanging and verifying token

## 1251

Create business pipeline step - business creation failed

## 1284

Handle Social OAuth 2.0 callback - error reading user profile from User Url: <config.UserUrl>

## 1639

identityVerificationID session does not contain client_id

## 1656

Description
* Error with authorization response
* The request is missing a required parameter, includes an invalid parameter value, includes a parameter more than once, or is otherwise malformed

Troubleshooting
* Missing nonce on implicit flow
* Redirect URI might not be with an https://
* Only a localhost suffix can be used with http://
* Error with workflow

## 8809

Description:
* Received browser trust token is different from the one stored in the login session

Troubleshooting
* Start the auth flow again from the sign in or log in button
* The user is trying to start a session in a new tab, browser, or device when there's already a partially completed session in progress
* The user may have bookmarked the auth page when it's partially completed, instead of bookmarking the initial sign in or log in page
